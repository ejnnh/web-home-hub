# web-home-hub

## 项目简介

`web-home-hub` 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为 HTML 文件的结构化存储与静态展示空间。

适合存放各类前端演示、测试页面、小型工具页面或静态内容片段，便于版本管理与统一访问。

## 目录说明

```
web-home-hub/
├── pages/          # 存放所有独立 HTML 页面
├── assets/         # 公共资源文件（CSS、JS、图片等）
├── index.html      # 入口导航页（可选）
└── README.md       # 本文件
```

- **pages/**：每个子目录或文件对应一个独立的 HTML 页面，按功能或主题命名。
- **assets/**：所有页面共享的样式、脚本、字体或图片资源。
- **index.html**：可选的导航入口页面，用于快速访问所有已归档页面。

## 页面归档说明

- 每个 HTML 页面应保持自包含或仅依赖 `assets/` 下的公共资源。
- 页面之间无强依赖关系，可独立访问。
- 归档时请遵循以下约定：
  - 页面文件名使用小写英文，单词间以短横线连接（如 `example-tool.html`）。
  - 如有多个相关页面，可放入同一子目录（如 `pages/tools/`）。
  - 避免在页面中包含外部不可控资源（如外链脚本、跟踪代码等），以保持归档的稳定性。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 对应位置，如有需要可同步更新 `assets/` 公共资源。
- 修改页面：直接编辑 `pages/` 下的文件，保持版本历史可追溯。
- 删除页面：移除对应文件或目录，并确认无残留引用。
- 建议在提交信息中简要说明新增或修改内容，便于协作。

## 使用方式

1. 克隆仓库到本地或部署到静态托管服务。
2. 直接通过文件路径访问 `pages/` 下的任意 HTML 文件。
3. 也可通过 `index.html` 导航页浏览所有归档页面。
